# Flight Fare Forecasting

> Final project for the Time Series course  
> National Economics University (NEU), Vietnam ‚Äî 2026

An end-to-end machine-learning and time-series framework for predicting domestic Indian flight fares, built on the [Ease My Trip](https://www.kaggle.com/datasets/jillanisofttech/flight-price-prediction-dataset) dataset. The pipeline runs 12 sequential stages ‚Äî from raw data ingestion to route-level fare forecasting ‚Äî and produces 25 analytical figures, 10 structured reports, and a full academic report.

## Pipeline Stages

| Stage | Module | Description |
|-------|--------|-------------|
| 1 | `data_loader` | Load `Data_Train.csv`, validate schema |
| 2 | `preprocessor` | Parse duration (fixes 10√ó inflation bug), clean stops, one-hot encode |
| 3 | `eda` | 7 EDA figures ‚Äî fare distributions, airline/route breakdowns, monthly trend |
| 4 | `feature_engineering` | Expand 11 ‚Üí 45 features: cyclical time, route tier, duration components |
| 5 | `model_evaluator` | Random 80/20 split, 5-model baseline comparison |
| 6 | `feature_importance` | ExtraTreesRegressor split-importance ranking (Figure 10) |
| 7 | `model_evaluator` | Scatter/residual plots for all baseline models |
| 8 | `statistical_models` | TS suitability: ADF test, ACF/PACF, GARCH/SVAR/SARIMA/SARIMAX verdict |
| 9a | `statistical_models` | Temporal split ‚Äî train Mar‚ÄìMay, test Jun 2019 |
| 9b | `statistical_models` | SARIMAX ARIMA(0,1,2) on 18 weekly price aggregates |
| 9c | `statistical_models` | Residual diagnostics ‚Äî Ljung-Box, ACF/PACF on ML residuals |
| 9d | `statistical_models` | Hybrid LightGBM+SARIMAX (SARIMAX trend as 46th feature) |
| 10 | `tuner` | Optuna TPE ‚Äî 50 trials each for RF, XGBoost, LightGBM |
| 11 | `predictor` | Predict fares for `Test_set.csv` using best full-data model |
| 12 | `forecaster` | Daily fare forecasts per airline/route for a configurable month |

## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

Requires **Python 3.9+**.

### 2. Run the full pipeline

```bash
python pipeline.py
```

This runs all 12 stages end-to-end (~5‚Äì10 minutes depending on CPU). All outputs are saved under `outputs/`.

### 3. Pipeline options

```bash
# Faster run for development (fewer Optuna trials)
python pipeline.py --n-trials 10

# Forecast a different month (default is July 2019)
python pipeline.py --forecast-month 8

# Full custom forecast
python pipeline.py --forecast-month 8 --forecast-year 2019 --forecast-hour 7
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--n-trials` | 50 | Optuna trials per model |
| `--forecast-month` | 7 | Month to forecast (1‚Äì12) |
| `--forecast-year` | 2019 | Year to forecast |
| `--forecast-hour` | 9 | Synthetic departure hour (0‚Äì23) |

## Notable Methodology Points

- **Temporal split evaluation**: Training on March‚ÄìMay and testing on June 2019 drops LightGBM R¬≤ by 29 points versus random splitting, demonstrating that random-split metrics substantially overstate real-world performance.
- **TS suitability check**: GARCH, SVAR, SARIMA, and SARIMAX are formally assessed. With only 18 weekly observations, standalone SARIMAX achieves out-of-sample R¬≤ = ‚àí0.37. The hybrid architecture uses SARIMAX only as a trend signal.
- **Hybrid architecture**: Appending the SARIMAX weekly trend prediction as a 46th feature recovers 22.78 pp of the 29-point temporal performance penalty.
- **Bayesian tuning**: Optuna TPE search (50 trials √ó 3 models) provides consistent but modest gains (+0.024 avg R¬≤); model ranking is unchanged from defaults.
